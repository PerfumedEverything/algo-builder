# AculaTrade v2.0 — Техническое задание

## Бюджет: 10 000 руб.
## Срок: 2-3 недели

---

## Цель

Довести торговый движок до корректной работы — чтобы все цифры были точными как при реальных деньгах. Подключить Bybit для тестирования 24/7. Заменить сырой терминал на профессиональный TradingView.

## Приоритеты

1. **Корректность расчётов** — P&L, цены входа/выхода, бэктест
2. **Профессиональный терминал** — TradingView Advanced Charts (бесплатная лицензия)
3. **Bybit интеграция** — второй брокер, крипто 24/7
4. **Grid Trading** — если останется время

## Фаза 1: Корректность расчётов (КРИТИЧНО)

### Проблема
Стратегии показывают неточные данные, бэктест — заглушка, портфель показывает бюджет вместо реальных сумм.

### Задачи

#### 1.1 Портфельная аналитика — готовое решение
- Установить `@railpath/finance-toolkit` (активна, ноя 2025)
- Заменить кастомный код Sharpe/Sortino/VaR/max drawdown на вызовы из библиотеки
- Проверенные формулы вместо нашего кода — исключаем ошибки в расчётах

#### 1.2 Валидация свечей
- Установить `candlestick` (активна, июл 2025)
- Каждая свеча проходит `validateOHLC` перед расчётами: high >= low, volume >= 0, open/close в пределах high/low
- Битые данные от брокера отсеиваются до попадания в движок

#### 1.3 FIFO P&L калькулятор — тесты на 100%
- Текущий код `FifoCalculator` покрыть юнит-тестами на каждый сценарий:
  - Один вход, один выход
  - Несколько входов по разным ценам → один выход (средняя цена)
  - Частичное закрытие позиции
  - Полное закрытие + новый вход
  - Отрицательный P&L (убыток)
- Каждый тест с конкретными числами и ожидаемым результатом

#### 1.4 Бэктест — реальная оценка условий
- Текущий `BacktestService.runBacktest()` имеет заглушку: `getSignal` всегда возвращает "long"
- Заменить на реальную оценку: парсить `entryConditions`/`exitConditions` из стратегии, вычислять индикаторы через `IndicatorCalculator`, проверять условия на каждой свече
- Бэктест должен показывать результаты ЭТОЙ стратегии, а не просто TP/SL на рандомном входе

#### 1.5 Портфель — реальные суммы
- "Размер портфеля стратегий" = sum(реальных операций), не sum(strategy.budget)
- Strategy card "Позиция" = сумма реальных покупок, не бюджет
- Проверить что Phase 11 фиксы работают на проде

### Результат фазы 1
Все расчёты корректны. P&L совпадает при ручной проверке. Бэктест отражает реальную логику стратегии.

## Фаза 2: TradingView Advanced Charts

### Проблема
Текущий терминал на `lightweight-charts` — только свечи, без индикаторов на графике, без drawing tools. Выглядит сырым по сравнению с Veles.

### Задачи

#### 2.1 Получение лицензии
- Подать заявку на https://www.tradingview.com/advanced-charts/ (кнопка "Get the library")
- Условия бесплатной лицензии:
  - Публичный сайт — AculaTrade да
  - Не за paywall — регистрация бесплатна, да
  - Брендинг TradingView виден — добавим
- Дождаться одобрения и доступа к private npm/GitHub

#### 2.2 Замена графика
- Удалить `lightweight-charts`
- Установить TradingView Advanced Charts
- Создать компонент `TradingViewChart` вместо текущего `InstrumentChart`

#### 2.3 Datafeed API
- Реализовать `UDFCompatibleDatafeed` — адаптер, который отдаёт наши свечи в формате TradingView
- Для T-Invest: свечи через `BrokerService.getCandles()`
- Для Bybit (фаза 3): через `BybitBrokerProvider.getCandles()`
- Realtime updates через `subscribeBars` → наш price stream

#### 2.4 Что получаем из коробки
- 100+ индикаторов на графике (RSI, MACD, Bollinger, EMA, SMA...)
- 110+ инструментов рисования (трендлайны, фибоначчи, горизонтальные уровни...)
- 12+ типов графиков
- Мобильная оптимизация
- Мультиязычность (русский из коробки)

### Результат фазы 2
Терминал на уровне Veles Finance. Индикаторы и drawing tools работают.

## Фаза 3: Bybit Provider

### Проблема
T-Invest работает только в часы MOEX (пн-пт 10:00-23:50 МСК). Нельзя тестировать стратегии на выходных. Нет возможности сравнить с Veles.

### Задачи

#### 3.1 Установка SDK
- `npm i bybit-api` — готовый TypeScript SDK для Bybit v5 API
- REST клиент для ордеров, баланса, позиций
- WebSocket клиент для реалтайм цен и стакана

#### 3.2 BybitBrokerProvider
- Новый класс реализующий интерфейс `BrokerProvider`
- Методы:
  - `getPortfolio()` — баланс USDT + открытые позиции
  - `getCandles(instrument, interval, from, to)` — OHLCV свечи
  - `getOrderBook(instrument, depth)` — стакан
  - `placeOrder(instrument, side, quantity, price?)` — лимитный/рыночный ордер
  - `cancelOrder(orderId)` — отмена
  - `getPrice(instrument)` — текущая цена
- Маппинг Bybit типов → наши типы (Candle, Position, Order)

#### 3.3 Реалтайм стриминг
- WebSocket подписка на тикеры (цены)
- WebSocket подписка на orderbook
- Интеграция с нашим price-stream-worker (pub/sub через Redis)

#### 3.4 Мульти-брокер UI
- Переключатель T-Invest / Bybit в настройках пользователя
- Терминал показывает крипто-пары при выборе Bybit (BTC/USDT, ETH/USDT...)
- 24/7 режим — без проверок сессий MOEX
- Точность цен до 8 знаков
- AI промпты адаптированы под крипто

### Результат фазы 3
Можно торговать на Bybit через AculaTrade. Стратегии работают 24/7. Данные идут через тот же движок что T-Invest.

## Фаза 4: Grid Trading

### Проблема
Нет сетки ордеров — основного инструмента крипто-трейдинга, который есть у Veles.

### Задачи

#### 4.1 GridTradingService
- Алгоритм из OpenTrader (open-source), адаптированный под наш Service Layer
- Расчёт уровней сетки по заданному диапазону
- Управление жизненным циклом ордеров (pending → filled → replace)
- P&L расчёт для сетки

#### 4.2 UI
- Новый тип стратегии `GRID` в StrategyConfig
- Форма настройки: инструмент, диапазон цен (от-до), количество уровней, объём на ордер
- Отображение сетки на графике TradingView (горизонтальные линии на уровнях)
- Мониторинг: исполненные ордера, текущие лимитники, P&L

#### 4.3 AI интеграция
- AI анализирует волатильность инструмента и предлагает параметры сетки
- "Для BTC/USDT рекомендую диапазон $62,000-$68,000, 12 уровней, $50 на ордер"

### Результат фазы 4
Grid trading работает на Bybit. Можно сравнивать с Veles на тех же стратегиях.

## Фаза 5: Сравнение с Veles + полировка

### Задачи

#### 5.1 Параллельное тестирование
- Выбрать 2-3 инструмента (BTC/USDT, ETH/USDT)
- Настроить одинаковые grid-стратегии на AculaTrade и Veles
- Запустить параллельно на реальном рынке (минимальные суммы, $10-50)
- Логировать каждую сделку с обеих сторон

#### 5.2 Анализ расхождений
- Сравнить: количество сделок, цены исполнения, P&L
- Для каждого расхождения — найти root cause в нашем коде
- Пофиксить

#### 5.3 Повторный прогон
- Запустить снова после фиксов
- Результаты должны совпадать с Veles (допуск ±1% по P&L из-за тайминга)

### Результат фазы 5
Движок валидирован. Уверенность что расчёты корректны. Можно применять те же фиксы к T-Invest.
