# Requirements: AculaTrade v2.0

**Defined:** 2026-03-28
**Core Value:** Корректный торговый движок с точностью реальных денег. Профессиональный терминал. Мульти-брокер (T-Invest + Bybit). AI-помощник для трейдинга.
**Spec:** .planning/v2.0-SPEC.md

## Phase 1: Корректность расчётов

### Портфельная аналитика
- [ ] **CALC-01**: Sharpe, Sortino, VaR, max drawdown рассчитываются через `@railpath/finance-toolkit`, не кастомным кодом
- [ ] **CALC-02**: Корреляционная матрица рассчитывается через библиотечную функцию
- [ ] **CALC-03**: Результаты библиотечных расчётов совпадают с ручным расчётом на 3 тестовых наборах данных

### Валидация свечей
- [ ] **CALC-04**: Каждая свеча проходит `validateOHLC` перед любыми расчётами (high >= low, volume >= 0)
- [ ] **CALC-05**: Битые свечи от брокера логируются и исключаются из массива данных
- [ ] **CALC-06**: Тесты подтверждают фильтрацию 3 типов битых свечей

### FIFO P&L
- [ ] **CALC-07**: FIFO P&L калькулятор покрыт 100% юнит-тестами — 6 сценариев с конкретными числами
- [ ] **CALC-08**: Средняя цена входа корректна при множественных входах по разным ценам
- [ ] **CALC-09**: Unrealized P&L корректен при частичном закрытии позиции

### Бэктест
- [x] **CALC-10**: `BacktestService.runBacktest()` парсит entryConditions/exitConditions из стратегии
- [x] **CALC-11**: На каждой свече бэктеста вычисляются индикаторы и проверяются условия
- [x] **CALC-12**: Вход только когда ВСЕ entry conditions выполнены, выход когда ANY exit ИЛИ TP/SL
- [x] **CALC-13**: Тест: стратегия "RSI<30 вход, RSI>70 выход" на известных данных → точки входа/выхода совпадают

### Портфель — реальные суммы
- [ ] **CALC-14**: "Размер портфеля" = sum(реальных операций BUY), не sum(strategy.budget)
- [ ] **CALC-15**: Strategy card "Позиция" = initialAmount из операций
- [ ] **CALC-16**: P&L% = суммарный P&L / суммарные реальные вложения × 100
- [ ] **CALC-17**: Тест: стратегия бюджет 10000₽, реальная покупка 9780₽ → портфель показывает 9780₽

## Phase 3: Bybit Provider

- [ ] **BYBIT-01**: `bybit-api` установлен, `BybitBrokerProvider` реализует `BrokerProvider` интерфейс
- [ ] **BYBIT-02**: `getPortfolio()` возвращает баланс USDT + открытые позиции с unrealized P&L
- [ ] **BYBIT-03**: `getCandles()` возвращает OHLCV свечи, проходящие `validateOHLC`
- [ ] **BYBIT-04**: `placeOrder()` выставляет лимитный и рыночный ордер на Bybit
- [ ] **BYBIT-05**: `cancelOrder()` отменяет ордер

- [ ] **BYBIT-09**: Автореконнект при обрыве WebSocket

### Мульти-брокер UI

## Phase 4: Grid Trading

### Движок
- [ ] **GRID-01**: `GridTradingService` рассчитывает уровни сетки равномерно между границами
- [ ] **GRID-02**: При исполнении buy → автоматически выставляется sell на уровень выше, и наоборот
- [ ] **GRID-03**: P&L сетки = sum(profit от каждой пары buy-sell)
- [ ] **GRID-04**: При выходе цены за границы → уведомление пользователю
- [ ] **GRID-05**: Тест: симуляция 100 тиков → проверить количество сделок и P&L

### UI
- [ ] **GRID-06**: Новый тип стратегии `GRID` в StrategyConfig с параметрами (lowerPrice, upperPrice, gridLevels, amountPerOrder)
- [ ] **GRID-07**: Форма создания с визуализацией уровней на графике TradingView
- [ ] **GRID-08**: Мониторинг: ордера на графике (исполненные/ожидающие), P&L в реалтайме
- [ ] **GRID-09**: Кнопка остановки: отменяет все pending ордера, фиксирует результат

### AI
- [ ] **GRID-10**: AI анализирует волатильность и предлагает параметры сетки (диапазон, уровни, объём)

## Phase 5: Валидация через Veles
