    **Step 3: Add getMarkowitzOptimization() to PortfolioAnalyticsService:**

    Extract `_buildReturnSeries(userId, positions, days)` as a private method reusing the candle-fetch + `toReturns` logic from `getCorrelationMatrix`. This avoids double candle fetch when both are called.

    The `getMarkowitzOptimization(userId: string, days = 90)` method:
    1. Get portfolio via `this.broker.getPortfolio(userId)`
    2. Filter STOCK/ETF positions only (same as getCorrelationMatrix)
    3. If < 2 positions, return `null`
    4. Call `_buildReturnSeries(userId, positions, days)` to get return arrays
    5. Build covariance matrix using `sampleCovariance` from `simple-statistics` pairwise
    6. Compute annualized mean returns: `mean(returns) * 252`
    7. Run Monte Carlo optimization (10,000 iterations):
       - Generate random weight vectors, normalize to sum=1
       - Enforce constraints: all weights >= 0, no single weight > 0.4
       - If a weight > 0.4, redistribute excess equally among others and re-normalize
       - Compute portfolio return: `w^T * meanReturns`
       - Compute portfolio variance: `w^T * Sigma * w`
       - Compute Sharpe: `(portReturn - rfRate) / portVol` where rfRate = 0.16 (current CBR key rate approx)
       - Track best Sharpe portfolio
    8. For positions with < 20 return observations, lock at current weight (don't optimize)
    9. Call `this.broker.getInstruments(userId, 'STOCK')` once to build `Map<ticker, lotSize>`
    10. Generate rebalancing actions:
        - `totalValue = sum(pos.quantity * pos.currentPrice)`
        - For each position: `targetValue = optimalWeight * totalValue`, `delta = targetValue - currentValue`
        - `lotSize = lotMap.get(ticker) ?? 1`
        - `lotsNeeded = Math.round(delta / (pos.currentPrice * lotSize))`
        - Action: lotsNeeded > 0 → 'BUY', < 0 → 'SELL', === 0 → 'HOLD'
        - `valueRub = Math.abs(lotsNeeded) * pos.currentPrice * lotSize`
    11. Return `MarkowitzResult` with weights array, rebalancing actions, expectedReturn, expectedVolatility, sharpe, insufficientData flag

    Important: import `sampleCovariance, mean` from `simple-statistics` (already used in project via `sampleCorrelation`).
