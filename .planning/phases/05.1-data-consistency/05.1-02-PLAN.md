# Plan 05.1-02: Algovist Ticker — Price & Indicator Accuracy

## Goal
All instruments always use algovist ticker variant — prices AND indicator values match T-Invest/TradingView exactly.

## Context
Our bot shows SBER at 316.40₽, T-Invest shows 316.65₽ at the same moment. The difference (~0.25₽) is because we resolve the wrong instrument variant. The algovist variant matches T-Invest display prices. This also means ALL indicators (RSI, SMA, MACD, etc.) are calculated on wrong candle data — formulas are correct, but input data is from the wrong instrument variant.

## Tasks

### Task 1: Investigate T-Invest API instrument variants
**Files:** `src/server/providers/broker/tinkoff-provider.ts`

**What:** Add debug logging to `resolveTickerToUid()` to see ALL variants returned by `findInstrument()` for SBER. Identify which classCode/instrumentKind corresponds to the algovist variant (matching T-Invest prices).

**Steps:**
1. Temporarily log all instruments returned by `findInstrument({ query: "SBER" })`
2. Compare classCode values: TQBR, CETS, etc.
3. Identify the variant whose price matches T-Invest app
4. Check if the pattern holds for other tickers (GAZP, LKOH, etc.)

### Task 2: Enforce algovist variant in resolveTickerToUid
**Files:** `src/server/providers/broker/tinkoff-provider.ts`

**Fix:** Update priority in `resolveTickerToUid()` to prefer the algovist variant:
1. Change classCode preference based on Task 1 findings
2. Ensure ALL price-related calls go through this single resolver

### Task 3: Enforce algovist variant in price-stream-worker
**Files:** `scripts/price-stream-worker.ts`

**Fix:** Apply same resolution logic as Task 2 in `resolveAll()` method.

### Task 4: Verify price accuracy
**What:** After fix, compare our price vs T-Invest for SBER, GAZP, LKOH — must match exactly.

### Task 5: Verify indicator accuracy after algovist fix
**What:** After algovist fix deployed, compare our indicator values vs TradingView for the same instrument and timeframe.

**Steps:**
1. Open TradingView на SBER (5m таймфрейм)
2. Записать значения: RSI(14), SMA(20), EMA(12), MACD(12,26,9), Bollinger(20,2)
3. Запросить те же индикаторы через наш API/сервис для SBER 5m
4. Сравнить значения — допустимая погрешность < 0.5%
5. Повторить для GAZP, LKOH
6. Если расхождение > 0.5% — исследовать причину (количество свечей, timing, округление)

**Key check:** RSI должен совпадать с TradingView с точностью до 0.1-0.5 пунктов. Если расхождение больше — проблема в данных (свечи), не в формулах.
