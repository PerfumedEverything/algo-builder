## What Changed

### Task 1: getPaperPortfolioAction hardening

**Before:** `if (!instrument) continue` and `if (stats.totalOperations === 0) continue` silently dropped strategies. Empty `catch {}` swallowed price fetch errors.

**After:**
- Instrument missing → fallback to `"—"`, row still included
- Zero operations → row still included with zeroed stats
- Price fetch error → `console.error("[PaperPortfolio] Price fetch failed for ...")` + row included with `undefined` price
- 5 unit tests covering all edge cases

### Task 2: StrategyTriggerHandler hardening

**Before:** Rollback DB call (`await this.db.from("Strategy").update(...)`) was unprotected — if it threw, the error propagated up and could crash the checker cycle.

**After:**
- Rollback wrapped in its own `try/catch`
- Failure logs `[CRITICAL] Rollback failed for strategy X, stuck in state Y` — no throw
- `getLastBuyPrice` retried once on failure before logging error

### Task 3: Strategy card error handling

**Before:** `if (res.success) setOperations(res.data)` — fetch error silently resulted in empty operations list (same as "no operations").

**After:** `toast.error("Не удалось загрузить операции")` on fetch failure, users can distinguish error state from empty state.
