<decisions>
## Implementation Decisions

### Price Cache Staleness (CRITICAL)
- **D-01:** Remove 7-day lastprice fallback in getPrice() — return null instead of week-old stale data. Stale prices cause false triggers.
- **D-02:** Reduce PRICE_TTL from 600s (10min) to 120s (2min) for intraday accuracy. Strategies on 1m/5m candles need fresher prices.
- **D-03:** Increase Redis lock TTL from 5s to 10s to prevent lock expiry during slow network conditions causing double triggers.

### Ticker @ Suffix Cleanup
- **D-04:** SQL migration to strip @ suffix from all existing Strategy and Signal instrument fields: `UPDATE "Strategy" SET instrument = REPLACE(instrument, '@', '') WHERE instrument LIKE '%@'` (same for Signal table).
- **D-05:** Add cleanTicker() call in strategy-checker.ts and signal-checker.ts when reading instrument from DB, as defense against future dirty data.

### Paper Portfolio Reliability
- **D-06:** getPaperPortfolioAction must never silently skip strategies. If instrument is missing, include row with "—" for price/P&L and log warning. If price cache fails, show realized P&L only (not skip entire row).
- **D-07:** Fix empty catch block in price fetch — log error, continue with null price.
- **D-08:** Fix fetchData in paper-portfolio-view.tsx — wrap in useCallback with stable deps to prevent re-fetching on every parent render.

### Signal Checker Lock Gap
- **D-09:** Add Redis lock to signal-checker checkAll() method, same as checkByInstrument(). Currently only checkByInstrument uses locks — checkAll can double-trigger.

### DB Update Error Handling
- **D-10:** Wrap lastIndicatorValues persistence in try/catch in both strategy-checker.ts and signal-checker.ts. Log failure, continue with stale values (next cycle will retry naturally).

### Trigger Handler Resilience
- **D-11:** If strategy-trigger-handler rollback fails after operation recording failure, log CRITICAL error with strategy ID + current state. Don't throw — leave strategy in last-known state for manual intervention.
- **D-12:** Add retry (1 attempt) for getLastBuyPrice in P&L calculation before falling back to no-P&L message.

### Strategy Card Display
- **D-13:** Distinguish between "no operations" and "fetch error" in strategy-card.tsx expanded view. Show error toast on fetch failure.

### Claude's Discretion
- Exact retry timing/backoff for price fetch
- Log format for CRITICAL errors
- Whether to add monitoring metrics (out of scope for now)

</decisions>
