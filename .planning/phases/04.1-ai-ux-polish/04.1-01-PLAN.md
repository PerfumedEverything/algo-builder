**ai-wizard-dialog.tsx** (~140 lines):
Create the 3-step wizard dialog. Type definition:
```typescript
type WizardStep = 'analysis' | 'strategy' | 'form'

type AiWizardDialogProps = {
  open: boolean
  onOpenChange: (open: boolean) => void
  title: string
  analyzeAction: () => Promise<{ success: boolean; data?: string; error?: string }>
  initialInstrument?: string
  onSuccess: () => void
  onCreateSignal?: () => void
}
```

Constants: `const STEPS = ['Анализ', 'Стратегия', 'Настройка']` and `const STEP_INDEX: Record<WizardStep, number> = { analysis: 0, strategy: 1, form: 2 }`.

State:
- `step` = useState<WizardStep>('analysis')
- `analysisResult` = useState<string | null>(null)
- `analysisLoading` = useState(false)
- `analysisError` = useState<string | null>(null)
- `extractedStrategy` = useState<AiGeneratedStrategy | null>(null)
- `formRef` = useRef<StrategyFormHandle>(null)

On open (`useEffect` on `open`): if open is true, reset all state to initial (step='analysis', analysisResult=null, etc.) and call `handleAnalyze()`.

`handleAnalyze` function: set loading=true, error=null, result=null. Call `analyzeAction()`. On success, set `analysisResult`. On error, set `analysisError`. Set loading=false.

`handleRedo` function: call `handleAnalyze()` (it already resets state).

`handleProceedToStrategy` function: `setStep('strategy')`.

`handleStrategyGenerated` function (called by AiChat `onGenerated`): call `formRef.current?.setGeneralFields(...)` with extracted fields, then `setStep('form')`.

Dialog structure (single Dialog, no nesting):
- `DialogContent className="max-w-2xl w-[calc(100vw-2rem)] p-0 flex flex-col max-h-[85vh]"`
- `DialogHeader` with `WizardStepIndicator` below the title.
- Content area with `overflow-y-auto min-h-[400px]`.

Step rendering — use `hidden` class (NOT conditional rendering) to keep all steps mounted:

**Step 1 (analysis):** `<div className={step !== 'analysis' ? 'hidden' : undefined}>`
- Loading state: Loader2 spinner + "Анализирую..." text.
- Error state: red bordered div with error message.
- Result state: Markdown rendered result (same prose classes as current AiAnalysisButton).
- After result, show "Повторить анализ" ghost button with RefreshCw icon (per AIUX-03). Disabled while loading.
- After result, show action buttons section "Что дальше?":
  - "Построить стратегию по анализу" button (blue, calls handleProceedToStrategy).
  - If onCreateSignal provided: "Настроить сигнал по анализу" button (outline, calls `() => { onOpenChange(false); onCreateSignal() }`).
  - Wrap buttons in `flex flex-col sm:flex-row gap-2` (per AIUX-04 mobile overflow fix).

**Step 2 (strategy):** `<div className={step !== 'strategy' ? 'hidden' : undefined}>`
- Render `AiChat` with `initialContext={analysisResult}`, `onGenerated={handleStrategyGenerated}`, `onStrategyExtracted={setExtractedStrategy}`.
- Below chat, render `StrategyPreviewPanel` if `extractedStrategy` is not null.

**Step 3 (form):** `<div className={step !== 'form' ? 'hidden' : undefined}>`
- Render `StrategyForm` with `ref={formRef}`, `mode="create"`, `onClose={() => onOpenChange(false)}`, `onSuccess={onSuccess}`.
- Above form, a "вернуться к AI помощнику" ghost button that sets `step='strategy'`.

Named export `AiWizardDialog`.
