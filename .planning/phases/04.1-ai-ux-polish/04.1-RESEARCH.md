# Phase 4.1: AI UX Polish - Research

**Researched:** 2026-03-26
**Domain:** React wizard/stepper UI pattern, multi-step dialog, mobile overflow, timeframe prop-drilling
**Confidence:** HIGH

## Summary

Phase 4.1 is a pure UI/UX refactor of existing Phase 4 components. No new packages are required ‚Äî the project already has everything needed. The core task is restructuring `AiAnalysisButton` and `StrategyDialog` into a 3-step wizard flow contained in a single dialog.

The wizard pattern in this project requires no external library. The existing shadcn/ui `Dialog` and `Tabs` components are sufficient building blocks. Wizard state is a simple `useState<'analysis' | 'strategy' | 'form'>` with a step indicator rendered above the dialog content. The step indicator itself is pure Tailwind ‚Äî numbered circles connected by lines.

The timeframe bug is a data-flow issue: `AiAnalysisButton.analyzeAction` is a closure constructed in terminal page, but it calls `buildChartMessage()` which correctly reads `period`. The actual call to `analyzeWithAiAction("chart", ...)` passes a hardcoded `"1d"` string in the system prompt inside `AI_PROMPTS["chart"]` ‚Äî not in the action call itself. Verification of the actual prompt is needed during planning.

**Primary recommendation:** Implement wizard as a new `AiWizardDialog` component that replaces the current separate `AiAnalysisButton` dialog and opens `StrategyDialog` inline. No new npm packages. Step indicator via inline Tailwind. Mobile overflow fix via `flex-col` button layout already applied in Phase 4 ‚Äî verify if it regressed.

### Pattern 1: Controlled Step Machine

**What:** Wizard step is a discriminated union managed with `useState`. Each step renders a different content region. No routing ‚Äî all in one dialog.

**When to use:** 3-step linear flow where steps have dependencies (step 2 needs step 1 data).

**Example:**
```typescript
type WizardStep = 'analysis' | 'strategy' | 'form'

const [step, setStep] = useState<WizardStep>('analysis')
const [analysisResult, setAnalysisResult] = useState<string | null>(null)
const [extractedStrategy, setExtractedStrategy] = useState<AiGeneratedStrategy | null>(null)

// Step transitions
const handleAnalysisDone = (result: string) => {
  setAnalysisResult(result)
  // Do NOT auto-advance ‚Äî wait for user action
}

const handleProceedToStrategy = () => setStep('strategy')
const handleProceedToForm = (strategy: AiGeneratedStrategy) => {
  setExtractedStrategy(strategy)
  setStep('form')
}
```

### Pattern 2: Step Indicator (pure Tailwind, no library)

**What:** Numbered circles connected by lines. Active step is filled, completed steps have check or different fill, future steps are outline only.

**Example:**
```tsx
type WizardStepIndicatorProps = {
  steps: string[]
  current: number // 0-indexed
}

// Step indicator using Tailwind only ‚Äî no additional component library
const WizardStepIndicator = ({ steps, current }: WizardStepIndicatorProps) => (
  <div className="flex items-center gap-0">
    {steps.map((label, i) => (
      <React.Fragment key={i}>
        <div className="flex flex-col items-center gap-1">
          <div className={cn(
            "flex h-7 w-7 items-center justify-center rounded-full text-xs font-medium",
            i < current && "bg-primary text-primary-foreground",
            i === current && "bg-blue-600 text-white ring-2 ring-blue-600/30",
            i > current && "border border-border text-muted-foreground"
          )}>
            {i < current ? <Check className="h-3.5 w-3.5" /> : i + 1}
          </div>
          <span className={cn(
            "text-[10px] whitespace-nowrap",
            i === current ? "text-foreground font-medium" : "text-muted-foreground"
          )}>{label}</span>
        </div>
        {i < steps.length - 1 && (
          <div className={cn(
            "h-px w-12 mb-4 shrink-0",
            i < current ? "bg-primary" : "bg-border"
          )} />
        )}
      </React.Fragment>
    ))}
  </div>
)
```

### Pattern 5: Seamless Wizard ‚Äî Single Dialog, No Close/Reopen

**What:** Instead of `AiAnalysisButton` closing its dialog and calling `onCreateStrategy()` which opens `StrategyDialog`, the wizard stays in one dialog and advances to step 2 directly. The `aiAnalysisResult` that was stored in terminal page state for seeding `StrategyDialog.initialContext` moves into wizard-internal state.

**Key implication:** `StrategyDialog` on terminal page (`strategyDialogOpen` state) becomes for the manual "–°–æ–∑–¥–∞—Ç—å —Å—Ç—Ä–∞—Ç–µ–≥–∏—é" button only (no AI context). The wizard handles the AI-seeded path.

### Anti-Patterns to Avoid

- **Auto-advance on analysis complete:** Do not auto-navigate from step 1 to step 2 when analysis finishes. User must read the result and click "–ü–æ—Å—Ç—Ä–æ–∏—Ç—å —Å—Ç—Ä–∞—Ç–µ–≥–∏—é". Auto-advance is jarring.
- **Keeping two separate dialogs:** The whole point of AIUX-05 is to avoid the current close‚Üíopen flow. Do not keep `AiAnalysisButton`'s own dialog and just wire it to open `StrategyDialog` programmatically.
- **Tabs for wizard navigation:** Tabs allow clicking any step out of order. Wizard steps here are sequential and dependent ‚Äî use controlled state with disabled forward navigation, not Tabs.
- **Duplicating StrategyForm:** Do not copy StrategyForm into the wizard. Use it as-is via a ref.
- **Dialog inside Dialog:** Do not nest a Dialog inside another Dialog. The wizard is a single Dialog root.

## Common Pitfalls

### Pitfall 1: Dialog Content Height Instability
**What goes wrong:** Step 1 (analysis result, variable length text) is taller than Step 2 (chat, fixed height). When advancing, dialog jumps in height.
**Why it happens:** `max-h-[85vh]` with `overflow-y-auto` doesn't constrain step content to a fixed height.
**How to avoid:** Set a minimum height on the content region (`min-h-[400px]`) and keep `max-h-[85vh]` on the dialog. Each step fills the same space.
**Warning signs:** Dialog visually resizes when advancing steps.

### Pitfall 2: Timeframe Hardcoded in System Prompt
**What goes wrong:** AI analysis always describes "daily" candles even when user selected 1h.
**Why it happens:** `AI_PROMPTS["chart"]` system prompt says "analyze daily price action" or similar.
**How to avoid:** During planning, read `ai-prompts.ts` and remove any timeframe references from system prompt. The timeframe is already in the user message from `buildChartMessage`.
**Warning signs:** AI response says "—Å—É–¥—è –ø–æ –¥–Ω–µ–≤–Ω—ã–º —Å–≤–µ—á–∞–º" when user had 1h selected.

### Pitfall 3: Step 3 Form Not Receiving Extracted Strategy
**What goes wrong:** StrategyForm in Step 3 doesn't pre-fill with AI-generated data.
**Why it happens:** The `StrategyFormHandle.setGeneralFields()` imperative method requires the form to be mounted. If Step 3 content is only rendered when `step === 'form'`, it unmounts/remounts on each step transition, losing ref state.
**How to avoid:** Use `display: none` / `hidden` class to hide Step 3 content instead of conditional rendering (`{step === 'form' && ...}`). The form stays mounted and the ref stays valid. Apply extracted strategy in a `useEffect` when `step` becomes `'form'`.
**Warning signs:** Form fields are empty when reaching Step 3 despite AI generating strategy.

### Pitfall 4: Mobile Overflow in Action Buttons
**What goes wrong:** "–ü–æ—Å—Ç—Ä–æ–∏—Ç—å —Å—Ç—Ä–∞—Ç–µ–≥–∏—é" and "–ù–∞—Å—Ç—Ä–æ–∏—Ç—å —Å–∏–≥–Ω–∞–ª" buttons in step 1 overflow on small screens.
**Why it happens:** `flex-row gap-2` with two full-width buttons at `flex-1` overflows when button text is long in Russian.
**How to avoid:** Use `flex-col sm:flex-row` so buttons stack vertically on mobile. Current `ai-analysis-button.tsx` already has this at line 112 (`flex flex-col sm:flex-row gap-2`). Verify it wasn't lost during wizard refactor.
**Warning signs:** Horizontal scroll appears in dialog on 375px viewport.

### Pitfall 5: Redo Resets AiChat Messages
**What goes wrong:** Clicking "–ü–æ–≤—Ç–æ—Ä–∏—Ç—å –∞–Ω–∞–ª–∏–∑" in Step 1 is fine, but if user already advanced to Step 2 (strategy chat) and goes back, the AiChat messages reset.
**Why it happens:** Going back to Step 1 resets `step` state, and `AiChat` has `useState` for messages initialized from `initialContext`. If AiChat remounts, it re-runs the auto-send effect.
**How to avoid:** Use `hidden` for step panels instead of conditional rendering to prevent remounting. Guard `autoSentRef.current` (already in AiChat) against re-fire.
**Warning signs:** AiChat sends analysis context message twice after redo.
