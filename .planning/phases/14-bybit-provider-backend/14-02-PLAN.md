In `getPortfolio()` method of `TinkoffProvider`:

**Replace aggregate avg price calculation (lines ~163-165):**
```
// BEFORE:
const fifoSummary = FifoCalculator.calculateSummary(ops, currentPrice)
const averagePrice = fifoSummary.totalQuantity > 0 ? fifoSummary.avgPrice : apiAvgPrice

// AFTER:
const avgPriceFifo = toNumber(p.averagePositionPriceFifo)
const averagePrice = avgPriceFifo > 0 ? avgPriceFifo : toNumber(p.averagePositionPrice)
```

**Replace manual P&L calculation (lines ~166-168):**
```
// BEFORE:
const cost = averagePrice * quantity
const yieldAbs = (currentPrice - averagePrice) * quantity
const yieldPct = cost > 0 ? (yieldAbs / cost) * 100 : 0

// AFTER:
const yieldAbs = toNumber(p.expectedYieldFifo) || (currentPrice - averagePrice) * quantity
const cost = averagePrice * quantity
const yieldPct = cost > 0 ? (yieldAbs / cost) * 100 : 0
```

**Replace portfolio-level yield percentage (lines ~142-144):**
```
// BEFORE:
const expectedYieldPct = totalAmount > 0
  ? (expectedYieldAbs / (totalAmount - expectedYieldAbs)) * 100
  : 0

// AFTER:
const expectedYieldPct = toNumber(res.expectedYieldRelative) ||
  (totalAmount > 0 ? (expectedYieldAbs / (totalAmount - expectedYieldAbs)) * 100 : 0)
```
Note: Check at runtime whether `expectedYieldRelative` is a fraction (0.05 = 5%) or percentage (5.0 = 5%). If fraction, multiply by 100.

**Keep FifoCalculator ONLY for lots:**
```
const fifoSummary = FifoCalculator.calculateSummary(ops, currentPrice)
// Use fifoSummary.lots for per-lot breakdown ONLY
lots: fifoSummary.lots,
```

**Remove FifoCalculator from aggregate price/P&L flow** — it should only appear once, to compute `lots`.

**Switch getOperations to getOperationsByCursor:**
Replace the operations fetch in the Promise.all:
```
// BEFORE:
client.operations.getOperations({
  accountId,
  from: yearAgo,
  to: new Date(),
  state: OperationState.OPERATION_STATE_EXECUTED,
})

// AFTER:
client.operations.getOperationsByCursor({
  accountId,
  from: yearAgo,
  to: new Date(),
  operationTypes: [OperationType.OPERATION_TYPE_BUY, OperationType.OPERATION_TYPE_SELL],
  state: OperationState.OPERATION_STATE_EXECUTED,
  limit: 1000,
})
```

Adjust response parsing: `getOperationsByCursor` returns `{ items, hasNext, nextCursor }` instead of `{ operations }`. Map `items` to `PositionOperation[]`. Each `OperationItem` has: `figi`, `type`, `payment`, `price`, `quantity`, `date`, `yield` (realized P&L).

Build opsByFigi from `opsRes.items` instead of `opsRes.operations`. The field mapping:
- `op.figi` stays same
- `op.type` uses OperationType enum values (same as before)
- `op.payment` → `toNumber(op.payment)` for amount
- `op.price` → `toNumber(op.price)` for price
- `op.quantity` → `op.quantity` (already number)
- `op.date` → `op.date` (already Date)

If `hasNext === true` for the first page (>1000 ops in year), log a warning but don't paginate for MVP — one page covers most accounts.
