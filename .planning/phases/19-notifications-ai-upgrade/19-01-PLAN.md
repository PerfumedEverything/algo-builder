    **1. Notification deduplication (NOTIF-01):**

    In `signal-trigger-handler.ts`:
    - Import `redis` from `@/lib/redis`
    - At the start of `handle()`, before the triggerCount update, attempt a Redis SET NX:
      ```
      const lockKey = `notif:signal:${signal.id}:${signal.triggerCount + 1}`
      const acquired = await redis.set(lockKey, "1", "EX", 300, "NX")
      if (!acquired) return // another process already handling this trigger
      ```
    - This prevents both cron and realtime from sending the same notification

    In `strategy-trigger-handler.ts`:
    - Same pattern: `const lockKey = `notif:strategy:${strategy.id}:${result.side}:${Date.now() / 60000 | 0}``
    - Use 60-second granularity (minute-level dedup) with `SET NX EX 120`

    **3. P&L fix — no quantity=1 fallback (NOTIF-04):**

    In `strategy-trigger-handler.ts` line ~94:
    - Change `const quantity = recordedQuantity > 0 ? recordedQuantity : 1` to just use `recordedQuantity`
    - If `recordedQuantity === 0`, skip the P&L calculation entirely (don't show incorrect P&L)
    - The corrected block:
      ```
      if (buyPrice > 0 && recordedQuantity > 0) {
        const pnlPerShare = result.price - buyPrice
        const totalPnl = pnlPerShare * recordedQuantity
        const pnlPercent = ((pnlPerShare / buyPrice) * 100).toFixed(2)
        const pnlSign = totalPnl >= 0 ? "+" : ""
        message += `\n\n💰 *P&L:* ${pnlSign}${totalPnl.toFixed(2)}₽ (${pnlSign}${pnlPercent}%)`
        message += `\n📊 Вход: ${buyPrice.toFixed(2)}₽ → Выход: ${result.price.toFixed(2)}₽`
      }
      ```

    **4. Position duration on exit (NOTIF-03):**

    In `operation-service.ts`:
    - Add method `getLastBuyOperation(strategyId: string): Promise<StrategyOperation | null>` that returns the full operation (not just price), so we get `createdAt` for duration calculation
      ```
      async getLastBuyOperation(strategyId: string): Promise<StrategyOperation | null> {
        const ops = await this.repo.findByStrategyId(strategyId)
        const lastBuy = [...ops].reverse().find((o) => o.type === "BUY")
        return lastBuy ?? null
      }
      ```

    - Add to message: `\n⏱️ Позиция: ${formattedDuration}`
    - Helper function `formatDuration(ms: number): string` inline in strategy-trigger-handler.ts
