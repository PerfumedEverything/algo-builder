# Phase 15.1 Plan 01: Grid Bug Fixes Summary

**TDD regression suite covering 6 critical grid bugs: double P&L, wrong quantity formula, missed FILLED buy pairing, wrong counter qty, missing validation, and missing SL/TP enforcement — all fixed with 93 passing tests**

- **Completed:** 2026-03-28T13:10:00Z

## Accomplishments

- Fixed critical BUG-01 double P&L: each DB row now gets its own P&L contribution, not the total tick P&L
- Fixed BUG-02 P&L formula: cost uses pairedBuy.quantity * filledPrice, revenue uses level.quantity * currentPrice
- Fixed BUG-03 paired buy lookup: service now fetches ALL orders (FILLED + PENDING) so engine can find FILLED buys
- Fixed BUG-04 counter qty: counter SELL placed with filled BUY's actual quantity, not the original sell level quantity
- Fixed BUG-06/07: calculateLevels now throws on invalid inputs (lowerPrice <= 0, upperPrice <= lowerPrice, gridCount < 2)
- Fixed BUG-08: removed unused computeStdDev function and void stdDev from GridAiService
- Fixed BUG-10: processPriceTick now enforces config.stopLoss and config.takeProfit by calling stopGrid

## Decisions Made

- Regression tests split into two files due to Vitest module mocking scope: engine tests cannot share a file with service tests that mock `@/lib/grid-engine`
- The 100-tick simulation P&L >= 0 assertion removed — with correct formula using buyQty for cost, P&L can legitimately be negative in asymmetric quantity scenarios
- Counter SELL quantity = filled BUY's quantity (what was actually purchased), ensuring the grid always sells exactly what it bought
