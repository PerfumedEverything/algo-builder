# Phase 15.1 Plan 02: Grid Strategy UI Integration Summary

Grid integrated as a first-class strategy type in /strategies page — GridStrategyCard, GridStrategyDialog, type-discriminated rendering with dedicated stats fetching.

## What Was Built

**GridStrategyCard** (`src/components/strategy/grid-strategy-card.tsx`):
- Renders grid-specific fields: range (lowerPrice — upperPrice), levels, distribution, fee rate, SL/TP if set
- Shows cycle stats (totalBuys/totalSells) and realizedPnl when gridStats available
- Dropdown menu: "Остановить" (ACTIVE only) and "Удалить" actions
- Exports `isGridConfig` type guard function for discriminated union checking
- Mirrors StrategyCard layout (GlowingEffect, same rounded-lg, same dropdown pattern)

**GridStrategyDialog** (`src/components/strategy/grid-strategy-dialog.tsx`):
- Two-step flow: Step 1 = instrument type + InstrumentSelect; Step 2 = GridForm
- Instrument selection uses existing InstrumentSelect component with price loading
- "Продолжить" button appears once ticker and price > 0 are loaded
- Back button returns to instrument selection
- Resets all state on close

**Strategies page** (`src/app/(dashboard)/strategies/page.tsx`):
- `isGridConfig(s.config)` discriminates Grid vs indicator strategies in all loops
- `getOperationStatsForStrategiesAction` called only with indicator strategy IDs
- Grid stats fetched via `Promise.allSettled(gridStrategies.map(s => getGridStatusAction(s.id)))`
- "Новая стратегия" button upgraded to DropdownMenu with "Индикаторная стратегия" and "Grid Bot" options
- GridStrategyCard rendered for Grid configs, StrategyCard for indicator configs
- GridStrategyDialog added with fetchData as onSuccess callback
