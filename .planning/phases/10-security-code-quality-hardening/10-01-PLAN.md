    **SEC-04: IDOR in operation-actions.ts**
    - In `getOperationStatsForStrategiesAction` (line 60-91): after `const userId = await getCurrentUserId()` on line 65, add ownership verification. Load user strategies: `const strategyService = new StrategyService()` then `const userStrategies = await strategyService.getStrategies(userId)`. Create a Set of user strategy IDs: `const ownedIds = new Set(userStrategies.map(s => s.id))`. Filter `strategyIds` to only include owned ones: `const validIds = strategyIds.filter(id => ownedIds.has(id))`. Use `validIds` instead of `strategyIds` for the rest of the function (in the instrumentMap loop on line 81 and the getStatsForStrategies call on line 86). Import StrategyService if not already imported.

    **SEC-07: Middleware bypass narrowing in middleware.ts**
    - On line 38-39, change the `isApiWebhook` check from:
      `request.nextUrl.pathname.startsWith("/api/webhooks") || request.nextUrl.pathname.startsWith("/api/signals")`
      to:
      `request.nextUrl.pathname.startsWith("/api/webhooks") || request.nextUrl.pathname === "/api/signals/check" || request.nextUrl.pathname === "/api/signals/check-instrument"`
    - This narrows the bypass from ALL `/api/signals/*` to only the two known cron endpoints.
