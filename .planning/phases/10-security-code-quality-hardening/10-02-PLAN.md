    **Create src/lib/rate-limit.ts:**
    - Import `redis` from `@/lib/redis`
    - Export `checkRateLimit(userId: string, action: string, maxRequests: number, windowSeconds: number): Promise<{ allowed: boolean; remaining: number }>`:
      - Key: `rate:${action}:${userId}`
      - Use `redis.incr(key)` then `redis.expire(key, windowSeconds)` on first increment (when count === 1)
      - Return `{ allowed: count <= maxRequests, remaining: Math.max(0, maxRequests - count) }`
    - Export `trackConnection(userId: string, action: string, maxConcurrent: number): Promise<{ allowed: boolean; release: () => Promise<void> }>`:
      - Key: `conn:${action}:${userId}`
      - Use `redis.incr(key)` to track active connections
      - Set TTL of 300s as safety net (auto-cleanup if release never called)
      - If count > maxConcurrent, decrement and return `{ allowed: false, release: async () => {} }`
      - Return `{ allowed: true, release: async () => { await redis.decr(key) } }`
    - Export `releaseConnection(userId: string, action: string): Promise<void>` — calls `redis.decr(key)` with floor at 0

    **Harden src/app/api/ai/chat/route.ts (SEC-02 + SEC-03 + SEC-05):**

    After auth check (line 32, after userId is obtained), add:

    1. **SEC-02 Rate limiting:** Import and call `checkRateLimit(userId, "ai-chat", 10, 60)`. If `!allowed`, return `new Response(JSON.stringify({ error: "Rate limit exceeded" }), { status: 429 })`.

    2. **SEC-03 Input validation:** After parsing body (line 34), add validation:
       - `if (!Array.isArray(messages) || messages.length === 0) return 400`
       - `if (messages.length > 50) return new Response(JSON.stringify({ error: "Too many messages" }), { status: 413 })`
       - `const totalChars = messages.reduce((sum, m) => sum + (m.content?.length ?? 0), 0)`
       - `if (totalChars > 50000) return new Response(JSON.stringify({ error: "Content too large" }), { status: 413 })`

    3. **SEC-05 Prompt injection (route side):** After body parsing, filter messages to only allow user/assistant roles:
       - `const safeMessages = messages.filter(m => m.role === "user" || m.role === "assistant")`
       - Use `safeMessages` instead of `messages` for the rest of the handler (assign to `enrichedMessages` and pass to provider)
