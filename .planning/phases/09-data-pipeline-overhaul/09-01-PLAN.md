       - `calculateRSI`: `const rsi = new RSI(period); candles.forEach(c => rsi.add(c.close)); return rsi.isStable ? Number(rsi.getResult()) : null`
       - `calculateSMA`: `const sma = new SMA(period); candles.forEach(c => sma.add(c.close)); return sma.isStable ? Number(sma.getResult()) : null`
       - `calculateEMA`: `const ema = new EMA(period); candles.forEach(c => ema.add(c.close)); return ema.isStable ? Number(ema.getResult()) : null`
       - `calculateMACD`: `const macd = new MACD(new EMA(fastPeriod), new EMA(slowPeriod), new EMA(signalPeriod)); candles.forEach(c => macd.add(c.close)); if (!macd.isStable) return null; const r = macd.getResult(); return { macd: Number(r.macd), signal: Number(r.signal), histogram: Number(r.histogram) }`
         CRITICAL: All 3 MACD constructor args must be EMA instances, NOT numbers
       - `calculateBollinger`: `const bb = new BollingerBands(period, stdDev); candles.forEach(c => bb.add(c.close)); if (!bb.isStable) return null; const r = bb.getResult(); return { upper: Number(r.upper), middle: Number(r.middle), lower: Number(r.lower) }`
       - `calculateATR`: `const atr = new ATR(period); candles.forEach(c => atr.add({ high: c.high, low: c.low, close: c.close })); return atr.isStable ? Number(atr.getResult()) : null`
       - `calculateStochastic`: `const stoch = new StochasticOscillator(period, 3, signalPeriod); candles.forEach(c => stoch.add({ high: c.high, low: c.low, close: c.close })); if (!stoch.isStable) return null; const r = stoch.getResult(); return Number(r.stochK)`
         CRITICAL: Result property is `stochK` not `k`
       - `calculateVWAP`: `const vwap = new VWAP(); candles.forEach(c => vwap.add({ high: c.high, low: c.low, close: c.close, volume: c.volume })); return vwap.isStable ? Number(vwap.getResult()) : null`
         Note: VWAP constructor takes no arguments
       - `calculateWilliamsR`: `const wr = new WilliamsR(period); candles.forEach(c => wr.add({ high: c.high, low: c.low, close: c.close })); return wr.isStable ? Number(wr.getResult()) : null`

    4. Verify file stays under 150 lines (project constraint). The rewritten methods are shorter than originals since streaming pattern is more concise.
