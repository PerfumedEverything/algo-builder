       - Takes `figi: string` parameter
       - Calls `getCurrentUserId()` first (security rule)
       - Creates BrokerService, fetches today's 1-minute candles:
         - `from`: today at 07:00 UTC (10:00 MSK session open) — compute using `MSK_OFFSET_MS` and MOEX main session start (10:00 MSK = 600 minutes): `const sessionStartUtc = new Date(today); sessionStartUtc.setUTCHours(7, 0, 0, 0)` (10 * 60 - MSK_OFFSET_MS/60000 = 420 min = 7h UTC)
         - `to`: now
         - `interval`: "1m"
       - If no candles returned, return `successResponse({ sessionOpen: 0, high: 0, low: 0, volume: 0 })`
       - Aggregate from candle array:
         - `sessionOpen = candles[0].open`
         - `high = Math.max(...candles.map(c => c.high))`
         - `low = Math.min(...candles.map(c => c.low))`
         - `volume = candles.reduce((sum, c) => sum + c.volume, 0)`
       - Return `successResponse({ sessionOpen, high, low, volume })`
       - Wrap in try/catch, return `errorResponse` on failure

    4. Also export a pure helper for unit testing:
       ```typescript
       export const aggregateSessionStats = (candles: { open: number; high: number; low: number; volume: number }[]): DailySessionStats => {
         if (candles.length === 0) return { sessionOpen: 0, high: 0, low: 0, volume: 0 }
         return {
           sessionOpen: candles[0].open,
           high: Math.max(...candles.map(c => c.high)),
           low: Math.min(...candles.map(c => c.low)),
           volume: candles.reduce((sum, c) => sum + c.volume, 0),
         }
       }
       ```
       This separates the pure aggregation logic from the server action wrapper, making it testable without mocking auth/broker.
