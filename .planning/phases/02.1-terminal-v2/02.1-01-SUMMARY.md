## What Was Built

- **Terminal types** (`src/core/types/terminal.ts`): `OrderBookLevel`, `OrderBookData`, `TopMover`, `PriceUpdate` — exported from core/types barrel
- **SSE route** (`/api/prices/stream`): Auth via `getCurrentUserId`, new Redis subscriber per connection, subscribes to `price-updates` channel, cleanup on abort/cancel, nginx-compatible headers (`X-Accel-Buffering: no`)
- **usePriceStream hook**: Client hook using `EventSource`, returns `Map<string, PriceUpdate>` keyed by instrumentId
- **getOrderBookAction**: Uses `TINKOFF_SYSTEM_TOKEN` (system token, not user token), calls `api.marketdata.getOrderBook`, converts Quotation format to `OrderBookData`
- **getTopMoversAction**: Calls `MOEXProvider.getTopMovers()`, public data with no auth
- **getOperationsByTickerAction**: Auth-guarded, fetches portfolio via `BrokerService`, returns operations for matched ticker
- **MOEXProvider.getTopMovers**: Fetches TQBR securities from MOEX ISS, joins marketdata+securities on SECID, filters nulls, sorts by LASTCHANGEPRCNT, 60s Redis cache
- **Unit tests**: 18 tests across 2 files — toNumber edge cases, mapOrderBookResponse spread calculation, parseTopMoversResponse null filtering/sorting/joining
