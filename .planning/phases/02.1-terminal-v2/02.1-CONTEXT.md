# Phase 2.1: Terminal v2 - Context

**Gathered:** 2026-03-24
**Status:** Ready for planning

<domain>
## Phase Boundary

Redesign the /terminal page from a simple chart viewer into a professional trading terminal. Replace lightweight-charts with TradingView Advanced Chart widget (free iframe embed). Add order book, realtime price bar, open positions, trade history, and top movers panels.

**NOT in scope:** Order placement (выставление заявок), active orders (активные заявки) — deferred to future phase.

</domain>

<decisions>
## Implementation Decisions

### Chart
- **D-01:** Use TradingView Advanced Chart Widget (free iframe embed) — NOT lightweight-charts. Provides all indicators, drawing tools, and realtime out of the box. MOEX tickers supported (MOEX:SBER, MOEX:GAZP, etc).
- **D-02:** Remove current lightweight-charts implementation from terminal page, replace entirely with TV widget.

### Layout
- **D-03:** Layout per Anton's wireframe (2026-03-24):
  - Top: price bar (ticker, price, change%, H/L, volume, bid/ask) — realtime
  - Middle: TradingView widget (left) + Order book/стакан (right) — side by side
  - Below chart: Open positions (left) + Trade history (right) — same row
  - Bottom: Top gainers (left) + Top losers (right) — all MOEX instruments for the day
- **D-04:** All blocks except Top gainers/losers are tied to the selected instrument. Top gainers/losers show data across ALL instruments.

### Order Book (Стакан)
- **D-05:** Simple table component showing bid/ask levels with volume. Data from T-Invest getOrderBook API. Not a custom-built complex widget — just display the API response as a styled table with bid/ask rows + spread.
- **D-06:** Спред shown below the order book.

### Realtime
- **D-07:** Prices must update in realtime. Price-stream-worker already has T-Invest WebSocket stream → Redis. Expose to frontend via SSE (Server-Sent Events) or polling.
- **D-08:** Price bar and order book should update live without page refresh.

### Top Movers
- **D-09:** Top gainers and top losers from MOEX ISS API (`/iss/engines/stock/markets/shares/boards/TQBR/securities.json`) sorted by LASTCHANGEPRCNT. Show top 5-10 in each direction.

### Positions & History
- **D-10:** Open positions shows user's current portfolio positions with realtime P&L (reuse existing portfolio data).
- **D-11:** Trade history shows operations for the currently selected instrument (reuse existing getOperationsAction).

### Claude's Discretion
- Exact SSE vs polling implementation for realtime
- Number of items in top movers (5 or 10)
- Order book depth (5 or 10 levels per side)
- Responsive/mobile layout approach

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- `price-stream-worker.ts`: Already streams T-Invest prices via WebSocket → Redis pub/sub. Can be extended with SSE endpoint.
- `PriceCache`: Redis-backed price cache with TTL, can serve realtime prices.
- `OperationService.getStats()`: P&L calculation for positions — reuse for positions panel.
- `getOperationsAction`: Fetches trade history by strategy — adapt for instrument-level history.
- `MOEXProvider`: Existing MOEX ISS integration — extend for top movers endpoint.
- `TinkoffProvider.getOrderBook()` may need to be added — check if T-Invest SDK supports it.

### Established Patterns
- Server Actions for data fetching from client components
- Redis caching with TTL for expensive API calls
- shadcn/ui components for all UI elements
- Tailwind utility classes for styling

### Integration Points
- `/terminal` route already exists — replace page content
- Sidebar navigation already has Terminal link
- Price worker already publishes to Redis `price-updates` channel
- Need new SSE API route (e.g., `/api/prices/stream`) for frontend realtime

</code_context>
