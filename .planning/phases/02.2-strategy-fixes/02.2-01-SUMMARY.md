# Phase 02.2 Plan 01: Foundation Fixes Summary

Null-safe IndicatorCalculator with period guards, Redis lock infrastructure in PriceCache (5m TTL fix), and DB migrations for CROSSES detection and positionState default.

## What Was Built

**IndicatorCalculator:** All 5 indicator methods (RSI, SMA, EMA, MACD, Bollinger) now return `null` when insufficient candle data is available instead of returning `0`. Added minimum candle count guards at the top of each method. Unit tests cover 21 scenarios.

**Strategy/Signal Checkers:** `getIndicatorValue` return type updated to `number | null`. `evaluateCondition` short-circuits to `false` when indicator returns `null` — insufficient data means the condition is not triggered, preventing false alerts.

**PriceCache:** Added `acquireLock` and `releaseLock` methods using Redis SET NX with 5-second TTL. 5m candle TTL corrected from 300s to 600s to provide 2x buffer over the candle refresh interval.

**DB Migrations:** Two idempotent SQL files ready for Supabase deployment — adds `lastIndicatorValues JSONB` to Strategy and Signal tables (needed for CROSSES detection in plan 02), and sets `positionState DEFAULT 'NONE'` with NULL backfill.
