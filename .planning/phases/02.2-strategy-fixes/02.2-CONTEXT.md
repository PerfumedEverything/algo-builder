# Phase 2.2: Strategy & Signal Pipeline Fix - Context

**Gathered:** 2026-03-25
**Status:** Ready for planning
**Priority:** CRITICAL — core product feature broken

<domain>
## Phase Boundary

Fix ALL strategy and signal checker bugs so that automated strategies work reliably with ANY condition type (PRICE, RSI, MACD, SMA, EMA, Bollinger, VOLUME, PRICE_CHANGE, SUPPORT, RESISTANCE). No new features — only fixes and hardening of existing pipeline.

</domain>

<decisions>
## Implementation Decisions

### CROSSES_ABOVE / CROSSES_BELOW (Bug #1 — CRITICAL)
- **D-01:** Implement real crossing detection: trigger ONLY when indicator transitions from below-above (or above-below) the threshold. Requires storing previous indicator value per strategy/signal.
- **D-02:** Add lastIndicatorValues JSON field to Strategy and Signal DB tables to track previous values for crossing detection.

### Indicator Calculator Validation (Bug #7 — CRITICAL)
- **D-03:** IndicatorCalculator must return null (not 0) when insufficient candle data. All callers must handle null as "cannot evaluate — skip".
- **D-04:** Add minimum candle count validation: RSI needs >= period+1, SMA/EMA needs >= period, MACD needs >= slow+signal, Bollinger needs >= period.

### Race Condition Dedup (Bug #2, #3)
- **D-05:** Replace .or() with proper atomic compare-and-swap. Use single .eq() with explicit NULL handling via DB default.
- **D-06:** Ensure positionState defaults to NONE in DB (not NULL) so .eq("positionState", "NONE") always works.

### Operation Recording Atomicity (Bug #11)
- **D-07:** If recordOperation fails after state change, ROLLBACK state change (set positionState back). Log the failure prominently.

### Error Visibility (Bug #10)
- **D-08:** Distinguish between "condition not met" and "system error" in check results. Add error boolean field to CheckResult type.

### Candle Cache Reliability (Bug #4, #5)
- **D-09:** Strategy checker should retry candle fetch once if cache miss AND broker fails.
- **D-10:** Increase 5m candle TTL from 300s to 600s (2x refresh cycle buffer).

### RESISTANCE/SUPPORT Edge Cases (Bug #14)
- **D-11:** Return null instead of Infinity/0 when no level found. Treat as "cannot evaluate".

### Concurrent Check Dedup (Bug #12)
- **D-12:** Add Redis-based lock per strategy/signal ID with 5s TTL. If lock exists, skip check.

### Unit Tests
- **D-13:** Every fix MUST have a unit test that reproduces the bug scenario BEFORE the fix, then verifies the fix works.

### Claude's Discretion
- Exact DB migration approach for lastIndicatorValues
- Redis lock key format
- Retry backoff timing

</decisions>
