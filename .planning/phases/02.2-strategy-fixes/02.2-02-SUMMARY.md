## What Was Built

**crossing-detector.ts:** New utility module (99 lines) housing all pure-function logic extracted from strategy-checker.ts — `getIndicatorKey` (deterministic key from indicator+params), `evaluateCrossing` (transition detection: prev < threshold AND current >= threshold), `getIndicatorValue`, `compareCondition`, `evaluateCondition`, `evaluateConditions`. This extraction enables strategy-checker.ts to stay under 150 lines.

**strategy-trigger-handler.ts:** New class (87 lines) extracted from handleTriggered — owns positionState compare-and-swap, recordOperation with rollback, P&L calculation, and Telegram notification. Receives `db` as constructor argument from StrategyChecker.

**strategy-checker.ts (149 lines):** Complete rewrite satisfying all plan requirements:
- Imports `evaluateConditions`, `getIndicatorValue`, `getIndicatorKey` from crossing-detector.ts
- `CheckResult.error?: boolean` distinguishes system errors from non-triggers
- `evaluateConditions` filters null results before AND/OR logic — null means "cannot evaluate", not false
- Entry guard uses `.eq("positionState", "NONE")` — no more `.or()` race condition
- `lastIndicatorValues` loaded from DB, updated after each check (enables crossing detection next cycle)
- Candle fetch retries once on failure with 1s delay
- Redis lock via `priceCache.acquireLock("strategy", id)` in both `checkAll` and `checkByInstrument` loops

**Tests:** Updated `strategy-checker-conditions.test.ts` with 16 new tests for `evaluateCrossing`, `getIndicatorKey`, and CROSSES_ABOVE/BELOW boundary conditions. Total: 30 tests, all passing.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed getConditionProgress null crash in signal-checker.ts**
- **Found during:** Task 2 — TypeScript caught `number | null` not assignable to `number` in `this.compare(current, ...)`
- **Issue:** `getConditionProgress` calls `this.getIndicatorValue` which returns `number | null`, then passes result directly to `this.compare(current, ...)` and `Math.round(current * 100)` — both crash on null
- **Fix:** Added null guard: `current !== null ? this.compare(...) : false`, and `current !== null ? Math.round(...) : null`
- **Files modified:** signal-checker.ts
- **Commit:** bead659

**2. [Rule 1 - Bug] Updated ConditionProgress type and signal-card.tsx null guard**
- **Found during:** Task 2 — cascade from signal-checker.ts fix
- **Issue:** `ConditionProgress.current: number` type no longer matched after null fix; `getProgressPercent` would divide by null
- **Fix:** Changed `ConditionProgress.current: number | null`, added `if (p.current === null || ...)` guard in `getProgressPercent`
- **Files modified:** signal-actions.ts, signal-card.tsx
- **Commit:** bead659
