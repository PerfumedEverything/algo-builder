# Phase 7.1 Plan 02: UI Components + Portfolio Page Rewire Summary

Health score card with 3 sub-score bars, diversification advice list, correlation warnings, and enhanced benchmark verdict replace Markowitz donuts and rebalancing table.

## What Was Done

### Task 1: Server Actions + AI Prompt Rewrite (04f626b)

Rewrote `getPortfolioAnalyticsAction` to use `Promise.allSettled` instead of `Promise.all` -- each metric (sector allocation, asset type, trade success, benchmark, dividends) now fails independently with sensible defaults. Added `getHealthScoreAction` that computes composite health score via `PortfolioHealthService`. Deleted `getMarkowitzOptimizationAction` entirely. ### Task 2: UI Components + Portfolio Page Rewire (e8d5398)

Created `HealthScoreCard` -- shows composite score (1-100) with colored number, level text, and 3 progress bars (diversification, risk, performance) each with detail lines. Created `DiversificationAdviceList` -- color-coded recommendations with Check/AlertTriangle/XCircle icons. Created `CorrelationWarnings` -- human-readable warnings below heatmap with correlation value badges. Enhanced `BenchmarkCard` -- added deposit comparison verdict section (verdictText, deposit delta, disclaimer). Deleted `markowitz-comparison.tsx` and `rebalancing-actions.tsx`. Rewired portfolio page layout: Row 1 = HealthScore + Benchmark + Dividends, Row 2 = Heatmap+Warnings + Sectors+AssetType, Row 3 = Concentration+Advice + TradeSuccess+PnlTable.
