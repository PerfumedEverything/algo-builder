    **1. Create migration** `supabase/migrations/20260329_grid_orders.sql`:

    ```sql
    CREATE TABLE IF NOT EXISTS grid_orders (
      id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
      grid_id       UUID NOT NULL REFERENCES "Strategy"(id) ON DELETE CASCADE,
      user_id       UUID NOT NULL,
      level_index   INT NOT NULL,
      price         DECIMAL(20, 8) NOT NULL,
      side          TEXT NOT NULL CHECK (side IN ('BUY', 'SELL')),
      quantity      DECIMAL(20, 8) NOT NULL,
      status        TEXT NOT NULL DEFAULT 'PENDING'
                    CHECK (status IN ('PENDING', 'FILLED', 'CANCELLED')),
      filled_at     TIMESTAMPTZ,
      filled_price  DECIMAL(20, 8),
      realized_pnl  DECIMAL(20, 8) DEFAULT 0,
      created_at    TIMESTAMPTZ DEFAULT NOW(),
      updated_at    TIMESTAMPTZ DEFAULT NOW(),
      UNIQUE (grid_id, level_index, side)
    );

    CREATE INDEX idx_grid_orders_grid_id ON grid_orders(grid_id);
    CREATE INDEX idx_grid_orders_status ON grid_orders(grid_id, status);

    ALTER TABLE grid_orders ENABLE ROW LEVEL SECURITY;
    CREATE POLICY "Users can manage own grid orders"
      ON grid_orders FOR ALL
      USING (user_id = auth.uid());
    ```

    Note: References `"Strategy"(id)` — check actual table name in existing migrations. The Strategy table name is PascalCase in Supabase.

    **2. Create `src/server/repositories/grid-repository.ts`:**

    Follow StrategyRepository pattern (class with `private async db()`).

    Methods:
    - `async createOrders(gridId: string, userId: string, orders: Array<{levelIndex, price, side, quantity}>): Promise<void>` — bulk insert all grid orders
    - `async getOrdersByGridId(gridId: string, userId: string): Promise<GridOrderRow[]>` — all orders for a grid
    - `async getPendingOrders(gridId: string, userId: string): Promise<GridOrderRow[]>` — only PENDING orders
    - `async fillOrder(gridId: string, levelIndex: number, side: string, filledPrice: number, realizedPnl: number): Promise<boolean>` — atomic update with `WHERE status = 'PENDING'`, returns false if already filled (race condition prevention per Pitfall 2)
    - `async activateCounterOrder(gridId: string, levelIndex: number, side: GridOrderSide, quantity: number): Promise<void>` — upsert a new PENDING order at the counter level
    - `async cancelAllPending(gridId: string, userId: string): Promise<number>` — cancel all PENDING, return count cancelled
    - `async getGridStats(gridId: string, userId: string): Promise<{totalBuys: number, totalSells: number, realizedPnl: number}>` — aggregate from filled orders
    - `async deleteByGridId(gridId: string, userId: string): Promise<void>` — cleanup

    Type `GridOrderRow`:
    ```typescript
    export type GridOrderRow = {
      id: string
      gridId: string
      userId: string
      levelIndex: number
      price: number
      side: 'BUY' | 'SELL'
      quantity: number
      status: 'PENDING' | 'FILLED' | 'CANCELLED'
      filledAt: string | null
      filledPrice: number | null
      realizedPnl: number
      createdAt: string
      updatedAt: string
    }
    ```

    CRITICAL for `fillOrder`: Use `.update({status: 'FILLED', filledAt: new Date().toISOString(), filledPrice, realizedPnl}).eq('grid_id', gridId).eq('level_index', levelIndex).eq('side', side).eq('status', 'PENDING')` — the `.eq('status', 'PENDING')` is the atomic guard against concurrent fills.
