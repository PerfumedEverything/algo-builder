    **1. `grid-levels-overlay.tsx`** — Custom hook `useGridLevelsOverlay`:

    ```typescript
    type UseGridLevelsOverlayProps = {
      chartRef: React.RefObject<IChartApi | null>
      seriesRef: React.RefObject<ISeriesApi<"Candlestick"> | null>
      levels: Array<{price: number, side: 'BUY' | 'SELL', status: 'PENDING' | 'FILLED' | 'CANCELLED'}>
      enabled: boolean
    }
    export const useGridLevelsOverlay = (props: UseGridLevelsOverlayProps) => { ... }
    ```

    Colors per RESEARCH Pattern 5:
    - FILLED: `#26a69a` (green)
    - SELL PENDING: `#ef5350` (red)
    - BUY PENDING: `#1976d2` (blue)
    - CANCELLED: `#757575` (gray)

    Uses `seriesRef.current.createPriceLine()` for each level. Cleans up on unmount or level change. LineStyle.Dashed, lineWidth=1, axisLabelVisible=true, title=`${side} #${index}`.

    **2. `grid-form.tsx`** — Grid creation dialog/panel:

    Uses react-hook-form + zodResolver for form state. Fields:
    - `name` (text input, required)
    - `lowerPrice` (number, required, > 0)
    - `upperPrice` (number, required, > lowerPrice)
    - `gridLevels` (number, min 3, max 100, default 10)
    - `amountPerOrder` (number, required, > 0, label "USDT per level")
    - `gridDistribution` (select: Arithmetic/Geometric, default Arithmetic) — per Bitsgap pattern, clear choice at top
    - `feeRate` (pre-filled 0.001 for Bybit 0.1%, editable)

    Live preview summary (updates on every field change, per our UX differentiator):
    - Total grid levels: N
    - Price step: (upper - lower) / (levels - 1), formatted
    - Profit per grid: step / lower * 100 - feeRate * 200, formatted as %
    - Total investment: amountPerOrder * gridLevels, formatted as USDT
    - Show grid level prices as a small list (scrollable if >10)

    On field change: compute levels via `GridEngine.calculateLevels(lower, upper, count, distribution)` and pass to `useGridLevelsOverlay` for live chart update. This is the KEY UX differentiator vs competitors.

    Submit button: "Запустить Grid Bot" — calls createGridAction, shows sonner toast on success/error.

    Per D-02: most user-friendly. Minimize required fields. Show computed metrics. Live chart preview.

    **3. `grid-monitor.tsx`** — Active grid monitoring panel:

    Props: `gridId: string`

    Uses polling (5s interval via useEffect) to call `getGridStatusAction(gridId)`.

    Displays:
    - P&L: realizedPnl formatted with currency (green if positive, red if negative)
    - Trades: totalBuys buys / totalSells sells
    - Orders list: table with columns [Level, Price, Side, Status, Filled At, P&L]
      - Color code rows: green=FILLED sell (profit), blue=PENDING, gray=CANCELLED
    - Stop button (red): calls stopGridAction, shows confirmation dialog first (per KuCoin pattern), then shows summary modal with final P&L breakdown per RESEARCH UX differentiators

    Per GRID-09: stop cancels all pending, shows final P&L.
    Per GRID-08: orders on chart use grid-levels-overlay with live status updates.
