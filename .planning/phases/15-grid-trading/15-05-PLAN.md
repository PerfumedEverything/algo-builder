    ```typescript
    export class GridAiService {
      static async suggestParams(params: {
        instrumentId: string
        instrument: string     // ticker e.g. "BTCUSDT"
        userId: string
        lookbackDays?: number  // default 30, options: 7, 30, 180 per Pionex pattern
      }): Promise<GridSuggestion>
    }

    export type GridSuggestion = {
      lowerPrice: number
      upperPrice: number
      gridLevels: number
      amountPerOrder: number
      gridDistribution: 'ARITHMETIC' | 'GEOMETRIC'
      feeRate: number
      reasoning: string          // AI explanation of why these params
      expectedProfitPerGrid: number  // % profit per completed grid cycle
      estimatedMonthlyTrades: number // estimated trades based on volatility
    }
    ```

    Implementation steps:
    1. Fetch candles for the lookback period using BrokerService.getCandles (1h interval for 30d = 720 candles)
    2. Compute ATR(14) using `trading-signals` library (same pattern as indicator-calculator.ts)
    3. Compute price statistics: current price, high, low, average, stddev over the period
    4. Determine grid range:
       - lowerPrice = currentPrice - 2 * ATR (rounded to instrument precision)
       - upperPrice = currentPrice + 2 * ATR (rounded to instrument precision)
       - Ensure range covers at least 4% of price (minimum viable grid)
    5. Determine gridLevels: based on range/ATR ratio, clamp to 5-30 range
       - levels = Math.round(priceRange / (ATR * 0.5)), clamped [5, 30]
    6. amountPerOrder: default 10 USDT (reasonable starting point for paper trading)
    7. feeRate: 0.001 (Bybit default 0.1%)
    8. expectedProfitPerGrid = (step / lowerPrice * 100) - (feeRate * 200)
    9. estimatedMonthlyTrades based on how many times price crossed ATR-sized moves in lookback

    10. Build reasoning string with DeepSeek (optional — if DeepSeek call fails, use templated reasoning):
        - Prompt: "Analyze {instrument} for grid trading. ATR(14)={atr}, range={low}-{high}, current={price}. Explain in 2-3 sentences why the suggested range of {lowerPrice}-{upperPrice} with {levels} levels is appropriate. Respond in Russian."
        - Use existing AI provider pattern (DeepSeek chat model, not reasoner)
        - Timeout 10s. If fails, reasoning = template: "Диапазон рассчитан на основе ATR(14) = {atr}. Охватывает 2 ATR от текущей цены в обе стороны."

    Per D-01: AI suggestion inspired by Pionex AI 2.0 (volatility-based range).
    Per RESEARCH "Don't Hand-Roll": use trading-signals ATR, not custom code.

    ```typescript
    export async function suggestGridParamsAction(params: {
      instrumentId: string
      instrument: string
      lookbackDays?: number
    }): Promise<ApiResponse<GridSuggestion>> {
      const userId = await getCurrentUserId()
      try {
        const suggestion = await GridAiService.suggestParams({
          instrumentId: params.instrumentId,
          instrument: params.instrument,
          userId,
          lookbackDays: params.lookbackDays,
        })
        return { success: true, data: suggestion }
      } catch (error) {
        return { success: false, error: error instanceof Error ? error.message : "Failed to suggest grid params" }
      }
    }
    ```
