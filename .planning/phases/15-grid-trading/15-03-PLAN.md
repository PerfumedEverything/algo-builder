    Actions (all start with `const userId = await getCurrentUserId()`):

    **createGridAction(params: { name: string, instrument: string, instrumentType: string, config: GridConfig, currentPrice: number }):**
    - Validate with Zod: lowerPrice > 0, upperPrice > lowerPrice, gridLevels >= 3 && <= 100, amountPerOrder > 0, feeRate >= 0 && <= 0.01
    - Validate: currentPrice >= lowerPrice && currentPrice <= upperPrice (must start in range)
    - Validate: totalInvestment = amountPerOrder * gridLevels (show in error if too large)
    - Call gridTradingService.createGrid({userId, ...params})
    - Return ApiResponse<{gridId: string}>

    **stopGridAction(gridId: string):**
    - Call gridTradingService.stopGrid(gridId, userId)
    - Return ApiResponse<{cancelledCount, stats}>

    **getGridStatusAction(gridId: string):**
    - Call gridTradingService.getGridStatus(gridId, userId)
    - Return ApiResponse with orders and stats

    **processGridTickAction(gridId: string, currentPrice: number):**
    - Call gridTradingService.processPriceTick(gridId, userId, currentPrice)
    - Return ApiResponse<GridTickResult>

    Zod schema for GridConfig validation:
    ```typescript
    const gridConfigSchema = z.object({
      type: z.literal('GRID'),
      lowerPrice: z.number().positive(),
      upperPrice: z.number().positive(),
      gridLevels: z.number().int().min(3).max(100),
      amountPerOrder: z.number().positive(),
      gridDistribution: z.enum(['ARITHMETIC', 'GEOMETRIC']),
      stopLoss: z.number().positive().optional(),
      takeProfit: z.number().positive().optional(),
      feeRate: z.number().min(0).max(0.01),
    }).refine(d => d.upperPrice > d.lowerPrice, { message: "Upper price must be greater than lower price" })
    ```

    Per security rules: every action calls getCurrentUserId() first, entity access filtered by userId.
