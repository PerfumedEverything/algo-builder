**Root cause (RCBF-01):** `handleQuickAction` in ai-chat.tsx calls `sendMessage("Да, создай эту стратегию", { forceCreate: true })` which triggers DeepSeek tool_call, yields a strategy chunk parsed into `pendingStrategy` inside `useAiStream`, fires `onStrategyExtracted` (preview update only), but never calls `onGenerated` — so the wizard never advances to `step: "form"`.

The fix has two parts:

1. **`src/hooks/use-ai-stream.ts`** — Change `sendMessage` return type from `Promise<void>` to `Promise<AiGeneratedStrategy | undefined>`. After the SSE loop and `finally` block completes, `return pendingStrategy`. Update the `UseAiStreamReturn` type accordingly. This surfaces the strategy to the caller without breaking the hook's abstraction (the hook doesn't know about wizard navigation).

2. **`src/components/strategy/ai-chat.tsx`** — In `handleQuickAction`, when `action.action === "CREATE"`:
   - Await the `sendMessage` call: `const strategy = await sendMessage("Да, создай эту стратегию", { forceCreate: true, context: instrumentContext })`
   - If `strategy` is defined, call `setFromAI(strategy.config)` then `onGenerated(strategy)` — the same two calls that `handleApply` makes.
   - This converges Path B (quick action) to the same endpoint as Path A (inline "Применить" button).

Do NOT add `onGenerated` logic inside the hook — keep wizard navigation in the component layer per Research pitfall #1.

**Root cause (RCBF-05):** `deleteStrategy` directly calls `repository.delete(id, userId)` without deactivating. If the strategy is ACTIVE, an in-flight StrategyChecker cycle could process it during deletion.

In `src/server/services/strategy-service.ts`, `deleteStrategy` method:

1. Before the existing `this.repository.delete(id, userId)` call, add:
   `await this.repository.update(id, userId, { status: "PAUSED" })`
   This ensures any concurrent StrategyChecker cycle sees PAUSED and skips it.

2. The existing Redis lock in StrategyChecker.checkAll prevents true races, so this is a defense-in-depth measure. No other layers need changes — StrategyChecker already filters by `status: "ACTIVE"`.

Keep the fix minimal — one line addition. No need to add broker cancel (paper trading only, no real orders).
