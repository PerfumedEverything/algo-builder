**Root cause (RCBF-07):** Strategy card shows "Позиция: 0.00₽" when `currentAmount = 0` due to price fetch failure, even though `holdingQty > 0` and `lastBuyPrice` is available. Strategies page "Размер портфеля стратегий" sums `config.risks.tradeAmount` (budget settings) instead of actual invested amounts from operations.

Three files:

1. **`src/server/actions/operation-actions.ts`** — In `getOperationStatsForStrategiesAction`, when computing stats for each strategy: if `currentAmount === 0` and `holdingQty > 0`, compute a fallback: `currentAmount = holdingQty * lastBuyPrice`. This uses cost basis as estimate when live price unavailable. The `OperationService.getStats` returns `lastBuyPrice` which is always available from operations. Apply this fallback in the action before returning to the client.

2. **`src/components/strategy/strategy-card.tsx`** — In the position display block (around line 129-133):
   - When `stats.holdingQty > 0` and `stats.currentAmount === 0`: display `stats.initialAmount` with label "Позиция (по цене входа)" instead of "Позиция: 0.00₽".
   - When `stats.currentAmount > 0`: keep existing "Позиция: {currentAmount} ₽" display.
   - This is a defense-in-depth UI fallback — the action-level fix (above) should prevent this case, but the UI should not show 0₽ for an open position regardless.

3. **`src/app/(dashboard)/strategies/page.tsx`** — In `portfolioSummary` useMemo (around line 59-64):
   - Replace the `totalPortfolio` computation that sums `config.risks.tradeAmount` with a sum of actual invested: `Object.values(opsStatsMap).reduce((sum, s) => sum + (s.initialAmount ?? 0), 0)`.
   - The `totalInitial` variable already computes this on line 65 — use it for `totalPortfolio` instead of the budget-based sum.
   - Remove or repurpose the `config.risks.tradeAmount` reduce — it shows budget, not actual investment.

Per Research pitfall #3: use `initialAmount` (sum of all BUY amounts) as fallback, not `lastBuyPrice * holdingQty`, because initialAmount accounts for multiple buys at different prices.
