**Root cause (RCBF-02):** `getDailySessionStatsAction` is hardcoded to fetch 1m candles from today's MOEX session open. The page only calls it in `handleInstrumentSelect`, not when period changes. So % is always daily.

Three files need changes:

1. **`src/server/services/session-stats.ts`** — Add `periodOpen` to the `DailySessionStats` type:
   ```
   export type DailySessionStats = {
     sessionOpen: number
     periodOpen: number   // first candle open of the selected period range
     high: number
     low: number
     volume: number
   }
   ```
   In `aggregateSessionStats`, set `periodOpen` same as `sessionOpen` (they're the same for intraday).

2. **`src/server/actions/chart-actions.ts`** — `getDailySessionStatsAction` accepts optional `period?: string` parameter.
   - For intraday periods (1m, 5m, 15m, 1h): keep current behavior — fetch from today's session open. Set `periodOpen = sessionOpen`.
   - For daily/weekly periods (1d, 1w): compute `from` date using the same `PERIOD_CONFIG` or candle range logic used for chart candles (look at `getCandlesForChartAction` for the pattern). Fetch one candle at that `from` date to get `periodOpen = candles[0].open`. Keep daily session stats (high/low/volume) from today's session.
   - Use `CandleInterval.CANDLE_INTERVAL_DAY` for 1d/1w period open lookup (single candle at period start).

3. **`src/app/(dashboard)/terminal/page.tsx`**:
   - Change `fetchDailyStats` to accept `period` parameter and pass it to `getDailySessionStatsAction(figi, period)`.
   - In `handlePeriodChange`, after `setPeriod(p)`, also call `fetchDailyStats(selectedInstrument.figi, p)` (same pattern as `handleInstrumentSelect`).
   - Change the `change` computation (around line 245-249) to use `dailyStats.periodOpen` instead of `dailyStats.sessionOpen`:
     ```
     const periodOpen = dailyStats?.periodOpen ?? dailyStats?.sessionOpen ?? 0
     const change = periodOpen > 0 ? ((currentPrice - periodOpen) / periodOpen) * 100 : 0
     ```

Per Research pitfall #2: for 1d period, `periodOpen` should be a point in time matching the chart range (e.g., 365 days ago for 1y equivalent). Look at how `getCandlesForChartAction` determines its date range and reuse that logic.

**Root cause (RCBF-04):** Entry notifications never include trade amount or quantity. Exit P&L is per-share (`result.price - buyPrice`) not total (`(result.price - buyPrice) * quantity`). The `formatStrategyNotification` function has no trade details parameter.

Two files:

1. **`src/server/services/notification-templates.ts`** — Extend `formatStrategyNotification` signature to accept optional trade details:
   ```
   type TradeDetails = {
     quantity: number
     amount: number
     totalPnl?: number
   }
   export const formatStrategyNotification = (
     strategy: StrategyRow, side: "entry" | "exit", ctx: EvalContext, trade?: TradeDetails
   ): string
   ```
   In the function body:
   - If `trade` is provided and side is "entry": append `\n\n📦 Куплено: ${trade.quantity} лот(ов) на ${trade.amount.toFixed(2)}₽`
   - If `trade` is provided and side is "exit": append `\n\n📦 Продано: ${trade.quantity} лот(ов) на ${trade.amount.toFixed(2)}₽`
   - Keep existing format lines intact.

2. **`src/server/services/strategy-trigger-handler.ts`** — In `handle` method:
   - After `recordOperation` succeeds (line ~55), capture the returned operation: `const op = await this.operationService.recordOperation(...)`.
   - Pass trade details to the message assignment: `message = formatStrategyNotification(strategy, side, ctx, { quantity: op.quantity, amount: op.amount })`
   - For exit enrichment (lines 67-86): compute total P&L as `(result.price - buyPrice) * op.quantity` instead of `result.price - buyPrice`. Update the P&L line accordingly:
     ```
     const totalPnl = (result.price - buyPrice) * op.quantity
     const pnlPercent = ((pnl / buyPrice) * 100).toFixed(2)  // pnl per share for %
     message += `\n\n💰 *P&L:* ${totalPnl >= 0 ? "+" : ""}${totalPnl.toFixed(2)}₽ (${pnlPercent}%)`
     ```
   - Verify that `recordOperation` returns the operation object (check operation-service.ts). If it returns void, need to capture quantity from input and compute amount.

Per Research pitfall #4: per-share % is correct for percentage, but absolute P&L must be `price_delta * quantity`.
